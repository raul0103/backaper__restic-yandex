# Backaper: логика работы

Этот проект описывает схему резервного копирования через связку **restic + rclone + Yandex Disk**.

## Как это работает

- `restic` отвечает за снимки (snapshot), дедупликацию, шифрование и восстановление.
- `rclone` даёт доступ к облаку Yandex Disk как к backend для `restic`.
- Данные берутся из локальной директории (обычно `~/`) и сохраняются в удалённый репозиторий restic.

Поток данных:

`локальная папка -> restic backup -> rclone backend -> Yandex Disk`

## Компоненты и роли

- `rclone`:
  - настраивает remote (например, `yandex`) через OAuth;
  - проверяет доступ к облаку (`rclone ls yandex:`);
  - используется restic в виде backend URL: `rclone:yandex:restic-repo`.
- `restic`:
  - инициализирует репозиторий (`restic init`);
  - делает инкрементальные бэкапы (`restic backup ~`);
  - хранит историю snapshot-ов (`restic snapshots`);
  - восстанавливает данные (`restic restore ...`).

## Базовая логика запуска

1. Настраивается `rclone` remote к Yandex Disk.
2. Задаются переменные:
   - `RESTIC_REPOSITORY=rclone:yandex:restic-repo`
   - `RESTIC_PASSWORD=<ваш_пароль>`
3. Один раз выполняется `restic init` для нового репозитория.
4. Выполняется первый бэкап: `restic backup ~`.
5. Последующие бэкапы запускаются той же командой — restic сохраняет только изменения.

## Почему это удобно

- Шифрование данных на стороне restic.
- Дедупликация и экономия места.
- История состояний (можно восстановить конкретный момент времени).
- Независимость: backend можно сменить, оставив ту же логику restic.

## Что важно не забыть

- Без `RESTIC_PASSWORD` расшифровать/восстановить данные невозможно.
- `restic init` нужен только один раз на новый путь репозитория.
- Восстановление лучше делать сначала в отдельную папку (`--target ~/restore-test`).
- OAuth-токены `rclone` и пароль restic нельзя публиковать.

## Файлы в этом репозитории

- `rclone.md` — установка и настройка rclone + Yandex Disk.
- `restic.md` — установка restic, бэкап, исключения и восстановление.
- `remote-orchestration.md` — restic на каждом сервере, запуск с control-машины по SSH, дамп БД.
- `central-backup.md` — альтернатива: одна машина, rsync по SSH, restic только на ней.
- `panel/` — **Laravel UI**: SSH-setup, поиск MODX, исключения, бэкап с дампом БД.

## Минимальный чек-лист

```bash
# проверить доступ к облаку
rclone ls yandex:

# проверить snapshot-ы
restic snapshots

# запустить бэкап
restic backup ~
```
